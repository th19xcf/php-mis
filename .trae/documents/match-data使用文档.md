# Match-Data 数据匹配使用文档

## 一、功能概述

Match-Data（数据匹配）是 MIS 系统中用于在两个数据表之间建立匹配关系的工具页面。常见应用场景包括：

- **收支认领**：将银行流水与记账凭证进行匹配
- **数据对账**：核对两个系统之间的数据一致性
- **关联标注**：在不同业务表之间建立关联关系

页面采用上下分栏布局，上方为 A 表数据、下方为 B 表数据，用户可在两侧分别选择记录，通过底部操作栏建立或撤销匹配关系。

## 三、核心功能操作指南

### 3.1 选择匹配条件

**操作步骤：**

1. 点击底部操作栏的 **「匹配条件」** 按钮
2. 在弹出的对话框中，勾选需要的匹配条件（可多选）
3. 点击 **「确定」** 完成选择

**说明：**
- 匹配条件由系统管理员在 `def_match_config` 表中预先配置
- 条件格式示例：`A.贷方金额=B.财务计收金额`、`A.对方名称=B.对方名称`
- 只有当所有选中的条件都满足时，两条记录才被视为"可匹配"
- **勾选记录前必须先选择匹配条件**，否则系统会弹出提示

### 3.2 勾选记录

**操作方式：**

- **单行勾选**：点击行首复选框
- **多行勾选**：按住 Ctrl 键点击多行，或使用 Shift 键进行范围选择
- **全选/取消全选**：点击表头复选框（仅对当前筛选结果有效）

**注意事项：**
- 未选择匹配条件时，系统会禁止勾选并弹出提示"请先选择匹配条件，再勾选记录"
- 选中的记录会以蓝色高亮显示
- A 侧和 B 侧需要分别勾选

### 3.3 显示数据筛选

点击底部 **「显示数据-全部」** 下拉按钮，可切换以下筛选模式：

| 筛选模式 | 说明 |
|---------|------|
| **全部** | 显示所有记录（默认） |
| **已匹配** | 只显示已建立匹配关系的记录 |
| **未匹配** | 只显示尚未匹配的记录 |
| **满足条件** | 只显示满足当前匹配条件的候选记录（包含已选记录） |

**「满足条件」模式说明：**
- 勾选 A 表记录后，B 表中满足匹配条件的记录会以黄色高亮显示
- 切换到「满足条件」模式后，B 表仅显示这些候选记录和已勾选记录
- 反之亦然（勾选 B 表后 A 表显示候选）

### 3.4 建立匹配

**操作步骤：**

1. 在 A 表勾选需要匹配的记录
2. 在 B 表勾选需要匹配的记录
3. 点击底部 **「建立匹配」** 按钮
4. 等待系统处理完成，页面会自动刷新

**匹配规则：**
- A 侧选中记录 × B 侧选中记录 = 建立多对多匹配关系
- 系统会根据配置的写入规则，将匹配信息写入双方数据表
- 写入方式可以是：字段值复制、UUID 生成、固定值等
- 一对多匹配时，目标字段会用英文分号 `;` 拼接多条值

**金额差额提示：**
- 如果配置了金额字段，底部会显示 A/B 两侧选中记录的金额合计及差额
- 金额不相等时以黄色警告形式提醒用户核对

### 3.5 撤销匹配

**操作步骤：**

1. 在 A 表或 B 表中勾选要撤销匹配的记录（任选一侧或两侧都选）
2. 点击底部 **「撤销匹配」** 按钮
3. 确认后系统会清除相关匹配数据
4. 页面自动刷新显示最新状态

**撤销范围：**
- 选中 A 侧记录：撤销这些 A 记录对应的所有匹配关系
- 选中 B 侧记录：撤销这些 B 记录对应的所有匹配关系
- 两侧都选：仅撤销选中 A 与选中 B 之间的匹配关系

## 四、表格辅助功能

### 4.1 快速检索

在表格右上角的搜索框中输入关键词，可实时过滤当前表格数据，支持对所有字段进行模糊搜索。

### 4.2 刷新数据

点击 **「刷新」** 按钮，重新从服务器加载当前侧的数据（不会影响另一侧）。

### 4.3 重置

点击 **「重置」** 按钮，清空当前侧的：
- 选中状态
- 列筛选条件
- 列排序设置

### 4.4 固定列

点击 **「固定列」** 按钮，可将常用列固定在表格左侧或右侧，横向滚动时保持可见。

### 4.5 字段选择

点击 **「字段选择」** 按钮，可自定义显示/隐藏表格列，根据需要调整列配置。

### 4.6 定位匹配

当选中记录后，点击 **「定位匹配」** 按钮，表格会自动滚动到第一条选中记录的位置。

### 4.7 匹配状态列

表格最右侧固定显示 **「匹配状态」** 列：
- **● 已匹配**（绿色）：该记录已建立匹配关系
- **○ 未匹配**（灰色）：该记录尚未匹配

## 五、行颜色含义

| 颜色 | 含义 | 场景 |
|------|------|------|
| 蓝色 | 已选中行 | 当前用户勾选的记录 |
| 黄色 | 候选匹配行 | 满足匹配条件的潜在匹配对象 |
| 浅绿色 | 已匹配行 | 已建立匹配关系的记录 |

## 六、配置说明（管理员）

### 6.1 功能配置（def_function）

在 `def_function` 表中新增一条匹配功能记录：

| 字段 | 说明 | 示例 |
|------|------|------|
| 功能编码 | 唯一功能编码 | 500001 |
| 功能名称 | 页面标题 | 收支认领（呼叫） |
| 模块名称 | 两个查询模块名，用英文逗号分隔 | 公司_财务_收支明细,公司_收入_记账_月明细 |
| 有效标识 | 是否启用 | 1 |

### 6.2 匹配配置（def_match_config）

在 `def_match_config` 表中配置匹配规则：

| 字段 | 说明 | 示例 |
|------|------|------|
| 功能编码 | 关联 def_function 的功能编码 | 500001 |
| A表名称 | A 侧物理数据表名 | 公司_财务_收支明细 |
| B表名称 | B 侧物理数据表名 | 公司_收入_记账_月明细 |
| A表主键 | A 表主键字段名 | GUID |
| B表主键 | B 表主键字段名 | GUID |
| 匹配条件 | 匹配条件，格式见下方 | A.贷方金额=B.财务计收金额;A.对方名称=B.对方名称 |
| A表写入B表 | A→B 的写入规则 | B.银行流水号=A.银行唯一流水号 |
| B表写入A表 | B→A 的写入规则 | A.记账表ID=uuid |

**匹配条件格式：**
```
A.<A表字段>=B.<B表字段>
```
多条条件用英文分号 `;` 分隔。

**写入指令格式：**
```
<目标表>.<目标字段>=<源>
```

源可以是以下类型：
- **字段引用**：`A.字段名` 或 `B.字段名`
- **UUID 生成**：`uuid`（自动生成 UUID v4）
- **字面量**：固定字符串值

一对多匹配时，来自多侧的字段值会自动用英文分号 `;` 拼接。

### 6.3 列配置（view_function）

匹配页面的列配置与普通工作台一致，通过 `view_function` 视图读取。

**可匹配字段类型（通过 `可匹配` 列标记）：**

| 值 | 含义 | 说明 |
|----|------|------|
| 1 | 主键字段 | 记录唯一标识，用于匹配定位 |
| 2 | 标签字段 | 选中记录预览时显示的名称 |
| 3 | 金额字段 | 用于计算金额差额 |
| 4 | 目标字段 | 存储匹配关系的字段 |

## 七、常见问题

### Q1：为什么点击勾选没有反应？
A：请先在底部点击「匹配条件」按钮选择至少一个匹配条件，否则系统会禁止勾选。

### Q2：「满足条件」模式下看不到数据？
A：需要先在另一侧勾选记录，系统会根据匹配条件计算出候选记录，然后切换到「满足条件」模式才能看到。

### Q3：为什么有的匹配记录颜色不一样？
A：
- 蓝色 = 当前选中的记录
- 黄色 = 满足匹配条件的候选记录（可匹配对象）
- 浅绿色 = 已建立匹配关系的记录

### Q4：建立匹配后数据在哪里？
A：匹配结果会写入物理数据表的指定字段中（由管理员配置），具体字段请咨询系统管理员。

### Q5：能撤销部分匹配吗？
A：可以。如果只选中一侧的记录，会撤销这些记录的所有匹配；如果两侧都选中，则只撤销选中记录之间的匹配关系。

## 八、技术架构

### 前端组件
- **index.vue**：页面入口，整合 A/B 表及底部操作栏
- **MatchTablePanel.vue**：表格面板组件（A/B 侧复用），含 AG Grid 表格、字段选择、固定列等
- **MatchBottomBar.vue**：底部操作栏，含显示筛选、匹配条件、建立/撤销匹配按钮
- **use-match-store.ts**：状态管理，处理数据加载、选中状态、候选计算等逻辑

### 后端接口
- **MatchApi::page()**：获取匹配页面初始数据（列配置、数据行、匹配条件等）
- **MatchApi::buildRelation()**：建立匹配关系
- **MatchApi::revokeRelation()**：撤销匹配关系

### 数据缓存
- 元数据表（`def_*`）使用长 TTL 缓存（3600s-86400s）
- 建立/撤销匹配后自动失效相关数据表缓存
