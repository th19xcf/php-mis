## 一、系统架构分析

### 1.1 现有系统架构

* **后端**: CodeIgniter PHP 框架

  * 控制器: `app/Controllers/XxxApi.php`

  * 路由: `app/Config/Routes.php`

  * 数据模型: `app/Models/Mcommon.php`

  * 错误码: `app/Constants/ApiCode.php`

* **前端**: Vue 3 + TypeScript

  * API服务: `src/service/api/xxx.ts`

  * 类型定义: `src/typings/api/xxx.d.ts`

  * 状态管理: `src/store/modules/xxx/`

  * 视图组件: `src/views/xxx/`

## 二、合同管理模块设计

### 2.1 业务流程图

```
[合同创建] → [合同审核] → [合同签署] → [合同归档] → [合同执行/终止]
     ↓           ↓            ↓           ↓
  [模板管理]  [审核流程]   [签署状态]   [文件管理]
                                              ↓
                                        [到期提醒/续签]
```

### 2.2 核心功能模块

| 模块   | 功能点           | 说明       |
| ---- | ------------- | -------- |
| 合同基础 | 合同创建、编辑、删除、查询 | CRUD基础操作 |
| 合同模板 | 模板定义、模板分类     | 提升创建效率   |
| 审核流程 | 多级审核、审核历史     | 流程可配置    |
| 签署管理 | 签署状态、手写签名、签章  | 追踪签署进度   |
| 文档管理 | 文件上传、版本管理、归档  | 安全存储     |
| 到期提醒 | 提醒设置、续签提示     | 主动通知     |

### 2.3 数据库表设计

```
def_contract          - 合同主表
def_contract_detail   - 合同明细表
def_contract_template - 合同模板表
def_contract_flow     - 合同流程表(审核记录)
def_contract_sign     - 签署记录表
def_contract_attachment - 合同附件表
def_contract_reminder - 提醒设置表
```

| 结束日期   | date          | 结束日期     |
| 付款方式   | varchar(50)   | 付款方式     |

### 4.2 合同状态枚举

| 状态码        | 状态名 | 说明    |
| ---------- | --- | ----- |
| DRAFT      | 草稿  | 初始状态  |
| PENDING    | 待审核 | 提交审核中 |
| APPROVING  | 审核中 | 审核流程中 |
| APPROVED   | 已审核 | 审核通过  |
| REJECTED   | 已拒绝 | 审核不通过 |
| SIGNING    | 签署中 | 签署流程中 |
| SIGNED     | 已签署 | 签署完成  |
| ARCHIVED   | 已归档 | 归档完成  |
| EXECUTING  | 执行中 | 合同执行中 |
| TERMINATED | 已终止 | 合同终止  |
| EXPIRED    | 已到期 | 合同到期  |

## 六、技术要点

### 6.1 状态流转控制

* 合同状态严格按照状态机流转

* 每个状态转换需要校验前置条件

* 记录完整的流程日志

### 6.2 权限控制

* 审核权限按流程节点配置

* 签署权限需要实名认证

* 删除权限仅限管理员

### 6.3 性能优化

* 列表分页加载

* 详情懒加载

* 树形结构虚拟滚动

### 6.4 用户体验

* 操作反馈即时提示

* 表单校验即时反馈

* 流程状态可视化展示
